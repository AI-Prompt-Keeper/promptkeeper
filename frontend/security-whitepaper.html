Security Whitepaper

Last updated: February 2026

Prompt Keeper is designed with security as a core requirement. This document describes our approach to encryption, transport security, and infrastructure controls.

1. Overview

We protect your prompts and API keys using industry-standard encryption at rest and in transit. Access is authenticated and least-privilege; data is stored in a secure, monitored environment.

2. Encryption at Rest

All stored data—including prompts, API keys, and metadata—is encrypted at rest.

  • Algorithm: AES-256 (industry standard for symmetric encryption).
  • Key management: Encryption keys are managed via a dedicated key management service (KMS). Keys are never stored in plaintext alongside the data they protect.
  • Scope: Full-disk or volume-level encryption is applied to all persistent storage; application-level encryption is used for sensitive fields (e.g., API keys) where required.
  • Key rotation: Keys are rotated according to a defined policy without requiring re-encryption of historical data where the design supports it.

3. Encryption in Transit

All data in transit between your systems and Prompt Keeper is encrypted.

  • TLS: All external endpoints are served over TLS 1.2 or TLS 1.3. Older protocols (SSLv3, TLS 1.0, TLS 1.1) are disabled.
  • Certificate management: Certificates are provisioned and renewed automatically; we use strong ciphers and prefer forward secrecy (e.g., ECDHE).
  • API and proxy traffic: Requests to our proxy (e.g., /v1/execute) and all responses are transmitted over HTTPS. We do not accept unencrypted HTTP for production traffic.
  • Upstream providers: When Prompt Keeper calls third-party LLM or API providers on your behalf, we use the provider’s TLS endpoints; we do not downgrade or strip encryption.

4. Access Control & Authentication

  • Access to the service is authenticated (e.g., API keys, OAuth, or SSO where offered).
  • Principle of least privilege: users and service accounts have only the permissions required for their role.
  • Administrative access to production systems is restricted, logged, and reviewed.

5. Infrastructure & Operations

  • Production workloads run in a secure, monitored environment with network segmentation where applicable.
  • Logging and monitoring are in place for security-relevant events (e.g., access, configuration changes).
  • We apply security patches and dependency updates according to our maintenance and risk policies.

6. Summary

Prompt Keeper uses AES-256 encryption at rest and TLS 1.2+ in transit for all sensitive data. Keys are managed securely, and access is controlled and audited. For questions or a detailed compliance questionnaire, contact your account team or security@promptkeeper.example.
